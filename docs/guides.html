Overview

This documentation explains how to create and manage Verifiable Documents, Pledge Contracts, Escrow Deposits, Document Registries, and Asset Paywalls on the platform.

üìë Verifiable Document

Steps

  1. Navigate to Create ‚Üí tap Create button.

  2. Upload file to:

    • IPFS
    • Arweave
    • Centralized host with direct download link

  3. Copy the file link.

  4. Fill the form:

    • Paste file link
    • Choose network

  5. Tap Create (top of screen).

  6. Confirm blockchain transaction.

  7. After minting ‚Üí go to History.

  8. Locate the document record.

  9. From More, select Initialize and Setup.

  10. Share the Verifiable Document DID.

üìú Pledge Contract

Issuer = Seller Must be created on the same network as the linked Verifiable Document.

Steps

  1. Navigate to Create ‚Üí tap Create.

  2. Fill the form:

    • Choose network

    • Select token

    • (Optional) Link to Verifiable Document:

      • Toggle Document Link
      • Enter Verifiable Document DID

  3. Tap Create (top of screen).

  4. Confirm transaction.

  5. After minting ‚Üí go to History.

  6. Locate pledge record.

  7. From More, select Initialize and Setup.

  8. Confirm transaction.

  9. Share Pledge Contract DID with buyer.

Fees & Conditions

  • 0% withdrawal fee
  • Other platform terms may apply

üîÑ Pledge Contract Status Guide

1. pending

Meaning: Service is initialized but not started yet.

  • Entered by: Default state.

  • Who can act: Seller (issuer).

  • Transitions to:

    • active ‚Üí seller starts service
    • canceled ‚Üí seller cancels before starting

  • Permissions:

    • ‚úÖ Rollback ‚Üí buyer
    • ‚úÖ Cancel ‚Üí seller
    • üö´ Withdraw

2. active

Meaning: Service has started and is ongoing.

  • Entered by: Seller starts service.

  • Who can act: Seller (execute), Buyer (dispute).

  • Transitions to:

    • executed ‚Üí seller executes service
    • disputed ‚Üí buyer disputes service

  • Permissions:

    • üö´ Rollback (unless expired)
    • üö´ Cancel
    • üö´ Withdraw

3. executed

Meaning: Seller marked service as delivered.

  • Entered by: Seller executes service.

  • Who can act:

    • Buyer ‚Üí confirm (completed)
    • Buyer ‚Üí dispute (disputed)
    • Seller ‚Üí withdraw after 5 days post-expiration (if no dispute)

  • Transitions to:

    • completed ‚Üí buyer confirms delivery
    • disputed ‚Üí buyer disputes service

  • Permissions:

    • üö´ Rollback
    • ‚úÖ Withdraw (after 5 days if no dispute)
    • üö´ Cancel

4. disputed

Meaning: Buyer raised a dispute.

  • Entered by: Buyer disputes service.

  • Who can act:

    • Buyer ‚Üí complete it
    • Seller ‚Üí cancel it

  • Transitions to:

    • completed ‚Üí by buyer
    • canceled ‚Üí by seller

  • Permissions:

    • üö´ Rollback
    • üö´ Withdraw
    • üö´ Cancel

5. completed

Meaning: Buyer confirmed delivery and acceptance.

  • Entered by: Buyer completes service.

  • Who can act: Seller ‚Üí withdraw.

  • Permissions:

    • ‚úÖ Withdraw (seller)
    • üö´ Rollback
    • üö´ Dispute
    • üö´ Cancel

6. canceled

Meaning: Seller canceled service before completion.

  • Entered by: Seller cancels.

  • Who can act: Buyer ‚Üí rollback.

  • Permissions:

    • ‚úÖ Rollback (buyer)
    • üö´ Withdraw
    • üö´ Execute/Complete

üìå Notes

  • Dispute only possible from active or executed.

  • Rollback allowed only when:

    • State is not active, or expired
    • State is not executed, completed, or disputed

  • Withdraw allowed only when:

    • State is completed, or
    • State is executed and 5 days passed with no dispute

  • No transitions allowed from completed, disputed, or canceled unless manual.

  • üîê Funds are locked until customer approves completion.

üìé Linked Document Rules

If linked to a Verifiable Document:

  • For executed ‚Üí seller must sign on-chain in the Document Contract.
  • For completed ‚Üí buyer must sign on-chain in the Document Contract.
  • For canceled ‚Üí Document Contract must be in suspended or revoked status.

üí∞ Escrow Deposit

Depositor = Customer/Buyer Stablecoin deposits require approval first.

Pre-requisite: Token Approval

  1. Navigate to Organize.
  2. Choose network and token.
  3. Set allowed approval amount.

Steps

  1. From Escrow Contract card ‚Üí tap Deposit Contract.

  2. Fill in the form:

    • Network
    • Token
    • Expiration time

  3. Tap Deposit (top of screen).

  4. Confirm transaction.

  5. Share Escrow DID with seller.

Fees & Conditions

  • Flat fee for deposit
  • 1% withdrawal platform fee
